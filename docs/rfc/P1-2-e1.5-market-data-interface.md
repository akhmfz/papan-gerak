# RFC E1.5 — Market Data Interface

> **Status**: Draft (menunggu approval)
> **Related**: [P1-2 Sprint E Experiment Design](./P1-2-sprint-e-experiment-design.md)
> **Tujuan**: Standarisasi format data untuk eksperimen kuantitatif, memisahkan data acquisition dari experiment engine.

---

## 1. Latar Belakang

Sprint E1 membuktikan bahwa experiment framework bekerja dengan data sintetis. Untuk E2, diperlukan data riil IDX80. Sumber data resmi adalah TradingView (environment yang sama dengan target deployment).

**Masalah**: Setiap kali data diimpor dari TradingView, format bisa berbeda (header, kolom, delimiter, encoding). Tanpa standarisasi, setiap eksperimen baru memerlukan kustomisasi import.

**Solusi**: Market Data Interface — sebuah layer abstraksi yang menerima CSV dengan format tetap dan mengeluarkan struktur data yang siap dikonsumsi oleh experiment engine.

---

## 2. Spesifikasi Format

### 2.1 File Format

| Atribut | Spesifikasi |
|---------|-------------|
| Format | CSV (Comma-Separated Values) |
| Encoding | UTF-8 |
| Separator | `,` (koma) |
| Decimal | `.` (titik) |
| Header | Wajib, lowercase |
| Baris baru | LF (`\n`) |
| Quote | `"` (double quote) untuk nilai mengandung koma |
| Timezone | UTC+7 (WIB) — sesuai Bursa Efek Indonesia |
| Missing values | Biarkan kosong (empty string), bukan `NA`/`N/A` |

### 2.2 Columns

#### Mandatory Columns

| Column | Type | Contoh | Validasi |
|--------|------|--------|----------|
| `ticker` | string | `BBCA` | Uppercase, 2-10 chars |
| `date` | string (YYYY-MM-DD) | `2026-01-05` | Tanggal valid, ascending per ticker |
| `baseline_score` | float [0,100] | `71.4` | Wajib, tidak boleh NaN |
| `mtf_score` | float [0,100] | `73.2` | Wajib, tidak boleh NaN |

#### Optional Columns

| Column | Type | Contoh | Validasi |
|--------|------|--------|----------|
| `timeframe` | string | `D` / `W` / `M` | Salah satu dari D/W/M |
| `trend` | float [0,100] | `80` | Range valid |
| `momentum` | float [0,100] | `74` | Range valid |
| `volatility` | float [0,100] | `62` | Range valid |
| `volume` | float [0,100] | `70` | Range valid |
| `mtf_direction` | int [-1,0,1] | `1` | -1 bearish, 0 mixed, 1 bullish |
| `mtf_penalty_pct` | float [0,100] | `2.5` | Persentase penalty MTF |
| `signal_base` | string | `BUY`/`SELL`/`HOLD` | Salah satu dari tiga nilai |
| `signal_mtf` | string | `BUY`/`SELL`/`HOLD` | Salah satu dari tiga nilai |

### 2.3 Contoh Template

```csv
ticker,date,timeframe,baseline_score,mtf_score,trend,momentum,volatility,volume,mtf_direction,mtf_penalty_pct,signal_base,signal_mtf
BBCA,2026-01-05,D,71.4,73.2,80,74,62,70,1,2.5,BUY,BUY
BBCA,2026-01-06,D,70.8,72.5,79,73,63,68,1,2.5,BUY,BUY
```

File template tersedia di: `tests/e2/templates/idx80-export.csv`

## 3. Pipeline

```
TradingView Export (.csv)
        │
        ▼
  importer.mjs
  │ - parse CSV
  │ - type coercion (string → number)
  │ - dedup by (ticker, date)
  │ - sort by ticker → date
  │ - output: Array<Object>
        │
        ▼
  validator.mjs
  │ - ticker format
  │ - date format & continuity
  │ - score range [0,100]
  │ - NaN detection
  │ - duplicate detection
  │ - gap detection (>5 hari)
  │ - output: ValidationReport
        │
        ▼
  experiment engine (KPI, scoring, report)
```

### 3.1 Importer (`tests/e2/importer.mjs`)

```js
import { importCSV } from './importer.mjs';
const data = importCSV('exports/idx80-2026-07.csv');
// data: [{ ticker, date, baseline_score, mtf_score, ... }, ...]
```

### 3.2 Validator (`tests/e2/validator.mjs`)

```js
import { validateDataset } from './validator.mjs';
const report = validateDataset(data);
// report.valid: boolean
// report.errors: string[]
// report.warnings: string[]
// report.stats: { baselineScore, mtfScore, delta, tickerCounts }
```

## 4. Aturan Dataset

1. **Satu file CSV = satu periode eksperimen.** Jika data mencakup 12 bulan, semua ticker harus memiliki cakupan tanggal yang sama.
2. **Satu baris = satu ticker × satu hari.** Tidak ada agregasi per baris.
3. **Missing values dibiarkan kosong.** Jangan diisi `0` atau `-1`.
4. **Ticker konsisten.** Jika di TradingView menggunakan `BBCA`, jangan gunakan `BBCA.JK` di baris lain.
5. **Timestamps tidak diperlukan.** Cukup kolom `date` YYYY-MM-DD.

## 5. Penggunaan

### Produksi (data riil)

```bash
# 1. Export CSV dari TradingView ke tests/e2/data/raw/
# 2. Import & validasi
node -e "
  const { importCSV } = require('./tests/e2/importer.mjs');
  const { validateDataset } = require('./tests/e2/validator.mjs');
  const data = importCSV('tests/e2/data/raw/idx80-export.csv');
  const report = validateDataset(data);
  console.log(report.valid ? '✅ VALID' : '❌ INVALID');
"

# 3. Jalankan eksperimen
node tests/e1/run-e1.mjs  # (setelah diupdate untuk menerima data riil)
```

### Testing

```bash
node tests/e2/run-e1.5.mjs
```

## 7. Risiko & Mitigasi

| Risiko | Dampak | Mitigasi |
|--------|--------|----------|
| Format CSV berbeda tiap export | Import gagal | Validator deteksi dan reject early |
| Ticker tidak konsisten (BBCA vs BBCA.JK) | Duplikat atau missing data | Validator peringatkan pola ticker |
| Gap data karena hari libur nasional | False positive gap detection | Validator toleransi 5 hari |
| Encoding tidak UTF-8 | Karakter rusak | Importer deteksi encoding mismatch |
| Score di luar range [0,100] | KPI tidak valid | Validator reject |
