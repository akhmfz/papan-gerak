# Contributing to Papan Gerak

1. Fork the repo
2. Create feature branch: `git checkout -b feat/your-feature`
3. Follow commit convention (see DEVELOPMENT.md)
4. Run `npm run ci` before committing
5. Open Pull Request

## Guidelines

- No silent changes — update CHANGELOG
- Every feature needs tests
- Maintain module line budgets
- Update docs for any user-facing changes
